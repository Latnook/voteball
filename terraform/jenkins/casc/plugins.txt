# Top-level plugins only -- jenkins-plugin-cli resolves dependencies itself, so this file stays
# readable and there is exactly one place to justify a plugin's existence.
#
# Comments are on their own lines, never trailing a plugin id: jenkins-plugin-cli parses each
# non-comment line as `artifact-id[:version]`, and a trailing `# why` would be read as part of it.
#
# Versions are DELIBERATELY unpinned. This host is rebuilt rarely and always wants the versions
# matching the Jenkins LTS it boots with; pinning here would mean a second, silently-stale list to
# maintain alongside the Jenkins version itself. The trade-off is that two builds of the same commit
# can install different plugin versions -- acceptable for a single-operator CI host, and the reason
# `jenkins-plugin-cli --list` output belongs in any bug report.
#
# Replaces the 94 plugins the setup wizard's "install suggested" left behind. Every entry below is
# traceable to something the Jenkinsfile or this host actually does.

configuration-as-code

job-dsl

workflow-aggregator
