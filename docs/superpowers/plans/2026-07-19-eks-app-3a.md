# EKS App Deploy — Plan 3a (deploy-core)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Get the Voteball app *running* on the EKS cluster and internally reachable — a restored RDS in the EKS VPC, the three container images in ECR, and the `charts/voteball` chart evolved to EKS-native so `helm upgrade --install` brings up frontend/backend/worker in the `devops-app` namespace, verified by a port-forwarded test vote landing in RDS. Public exposure + hardening (Ingress/ALB, NetworkPolicy, HPA, PDB, backup CronJob) is **Plan 3b**.

**Architecture:** RDS added to the `terraform-eks/` stack (restored from the k3s final snapshot, in the isolated DB subnets). Images built locally and pushed to the Plan-2 ECR repos with **git-SHA tags**. `charts/voteball` evolved in place: images become ECR URLs, the frontend moves to `nginxinc/nginx-unprivileged` (ALB will terminate TLS in 3b, so no in-pod 443), secrets come from Secrets Manager via an ESO **ExternalSecret**, non-secret config from a **ConfigMap**, schema bootstrap from a **Helm pre-install/upgrade migration Job** (running the `migrate.py` from Plan 1), and the `worker` ServiceAccount carries its IRSA annotation.

**Tech Stack:** EKS 1.34, Helm 3, `charts/voteball`, ECR, RDS Postgres 17, External Secrets Operator (Plan 2b), Docker, region `il-central-1`, account `590183895228`.

## Global Constraints

- **Namespace `devops-app`** for all app resources; install with `helm upgrade --install voteball charts/voteball -n devops-app --create-namespace`.
- **Non-root, uid 1000** backend/worker; frontend runs as the `nginx-unprivileged` uid **101**; all with `allowPrivilegeEscalation:false`, `capabilities.drop:[ALL]`, and `readOnlyRootFilesystem:true` + `emptyDir` for writable paths (backend/worker `/tmp`; frontend `/tmp` + `/var/cache/nginx`).
- **No `latest` tags** — images tagged with the **git SHA** (`git rev-parse --short HEAD`).
- **`sslmode=require`** to RDS (`DB_SSLMODE=require` in the ConfigMap).
- **Least-privilege IRSA:** only the `worker` SA is annotated with a role (`voteball-worker-irsa`); `backend` and `frontend` SAs carry **no** role. (The `backup` SA + CronJob are Plan 3b.)
- **No secret values in git or ConfigMaps** — the K8s Secret `app-secret` is materialized at runtime by ESO from Secrets Manager `voteball/app-secret`; a committed `secret-example.yaml` documents the shape with placeholders.
- **Verified live values (Plan 2 outputs):** ECR `590183895228.dkr.ecr.il-central-1.amazonaws.com/voteball-{backend,worker,nginx}`; worker IRSA `arn:aws:iam::590183895228:role/voteball-worker-irsa`; S3 bucket `voteball-rollups-590183895228`; SNS `arn:aws:sns:il-central-1:590183895228:voteball-notifications`; Secrets Manager `voteball/app-secret`.
- Run `terraform` from `terraform-eks/` in the main tree. **Commit and push to `master` as each task completes.** Never force-push.

**Pre-flight:** cluster live (`kubectl get nodes` → 2 Ready), ESO Running (Plan 2b), `kubectl` context = `voteball`.

# the isolated DB subnets (no NAT/IGW route) and only accepts 5432 from the EKS node security group.

    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_db_instance" "app" {
  identifier             = "${var.cluster_name}-eks-db"
  snapshot_identifier    = var.db_snapshot_identifier # restores votes; master password comes FROM the snapshot
  instance_class         = "db.t4g.micro"
  db_subnet_group_name   = aws_db_subnet_group.app.name

  publicly_accessible    = false
  skip_final_snapshot    = true # this is a throwaway demo DB (the k3s snapshot is the source of truth)

}

- [ ] **Step 1: Create the static `nginx.conf`**

Create `ansible-project/roles/frontend/files/nginx/nginx.conf`:

```nginx
# EKS: ALB terminates TLS and forwards plain HTTP here on 8080. No 80->443 redirect (the ALB does
# HTTP->HTTPS), no ssl_certificate. nginx-unprivileged listens on 8080 as uid 101.
server {
    listen 8080;
    server_name _;
    root /usr/share/nginx/html;

    # Serve "/" directly (no `index` directive) so it can't rewrite to /index.html and loop.
    location = / {
        try_files /index.html =404;
    }

    location = /index.html { return 301 /; }
    location = /results.html { return 301 /results; }
    location = /admin.html { return 301 /admin; }

    location / {
        try_files $uri $uri.html $uri/ =404;
    }

    # In-cluster DNS: "backend" resolves to the backend Service in the same namespace.
    location /api/ {
        proxy_pass http://backend:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $http_x_forwarded_proto;
    }
}
```

### Task 3: Build + push all three images to ECR

Build the three app images and push them to the Plan-2 ECR repos, tagged with the git SHA. This is the interim delivery path until CI (Plan 4).

- [ ] **Step 1: Create `scripts/build-push-ecr.sh`**

```bash
#!/usr/bin/env bash
# Build the three app images and push to ECR with a git-SHA tag. Run from the repo root.
# Requires: docker, aws CLI creds for account 590183895228, the Plan-2 ECR repos.
set -euo pipefail

REGION=il-central-1
ACCOUNT=590183895228
REGISTRY="${ACCOUNT}.dkr.ecr.${REGION}.amazonaws.com"
TAG="$(git rev-parse --short HEAD)"

echo "Logging in to ECR ${REGISTRY}"
aws ecr get-login-password --region "$REGION" | docker login --username AWS --password-stdin "$REGISTRY"

build_push() {
  local repo="$1" ctx="$2"
  echo "==> ${repo}:${TAG}"
  docker build -t "${REGISTRY}/${repo}:${TAG}" "$ctx"
  docker push "${REGISTRY}/${repo}:${TAG}"
}

build_push voteball-backend ansible-project/roles/backend/files/backend
build_push voteball-worker  ansible-project/roles/worker/files/worker
build_push voteball-nginx   ansible-project/roles/frontend/files/nginx

echo
echo "Pushed all three at tag: ${TAG}"
echo "Set image.tag=\"${TAG}\" in charts/voteball/values.yaml"
```
